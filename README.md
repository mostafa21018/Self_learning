
# 🚀 Laravel Backend Journey (Self-Learning)

## 🔷 Topics Covered (Study Notes)
هنا ملخص للمفاهيم الأساسية اللي ذاكرتها، ومربوطة بالملفات التفصيلية اللي رفعتها:

*   **[MVC Architecture](./MVC%20Architecture.md)**: 🏗️
    *   هو نظام لتقسيم الكود لـ 3 أجزاء مترابطة (Model - View - Controller).
    *   الهدف منه هو فصل الـ Business Logic عن الـ User Interface لضمان سهولة الصيانة والتطوير.
    *   **الـ Model**: بيتعامل مع قاعدة البيانات والبيانات.
    *   **الـ View**: هو الواجهة اللي بيشوفها المستخدم.
    *   **الـ Controller**: هو الوسيط اللي بيستقبل الـ Requests وبيربط بين الـ Model والـ View.

*   **[What is Laravel?](./Overview.md)**: 🛠️
    *   إطار عمل (Framework) مبني بلغة PHP ومفتوح المصدر.
    *   بيتبع نمط الـ MVC وبيوفر أدوات قوية زي **Eloquent ORM** و **Artisan CLI** و **Blade Template Engine**[cite: 3].
    *   بيعتمد على مكونات من Symfony وبيستخدم **Composer** لإدارة المكتبات.

*   **[PSR-FIG Standards](./PSR-FIG.md)**: 📏
    *   دي معايير عالمية (PHP Standards Recommendations) بتنظم طريقة كتابة الكود عشان المبرمجين والمكتبات يفهموا بعض.
    *   من أهمها **PSR-4** اللي بنستخدمه في الـ Autoloading للـ Clases.


## 🧠 Workflow with Obsidian
الملفات اللي في الـ Repo ده هي عبارة عن **Markdown files** مكتوبه باستخدام برنامج **Obsidian**:
1.  كل ملف بيحتوي على روابط داخلية عشان تربط المفاهيم ببعضها.
2.  تقدر تعمل `Clone` للـ Repo ده وتفتحه كـ `Vault` في Obsidian عشان تشوف التنسيق والروابط زي ما أنا عاملها بالظبط.
